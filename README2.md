# Проект "Таблица заказов" (Серверная Архитектура)

## **Обзор проекта**

Проект представляет собой интерактивную таблицу заказов. Главная цель — полная миграция логики обработки данных (пагинация, сортировка, фильтрация, поиск) на серверный API (http://localhost:3000/orders). Миграция повысила производительность и масштабируемость, оставив за клиентом только сбор параметров и отображение.

## **Реализованная функциональность**

Вся функциональность реализована через формирование запросов к API (query параметры).

_Пагинация_ (\_page, \_limit):

Переключение страниц и лимит строк (rowsPerPage) реализованы через формат запроса (?\_page=X&\_limit=Y).

_Сортировка_ (\_sort):

Кнопки сортировки (Date, Total) формируют параметр sort (field:direction). Сортировка автоматически сбрасывает пагинацию на page=1.

_Фильтрация_ (filter[field]):

Поля фильтра (Date, Customer, Total, Seller) отправляют параметры в формате ?filter[fieldName]=value.

Для Seller реализовано получение индексов (getIndexes()) и заполнение списка доступных опций.

_Поиск_ (search):

Поиск по всей таблице реализован через единственный параметр ?search=query. Локальная логика сравнения удалена.
